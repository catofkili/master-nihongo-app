# "一键填满"功能 - 红色警告提示更新

## 📅 更新时间
2026-06-16

## 🎯 更新目的
为"一键填满"功能添加醒目的红色警告提示，防止用户误操作导致学习进度丢失。

## ⚠️ 功能风险说明

### 高风险行为
"一键填满"功能的执行逻辑：
1. **先清空所有等级（N5-N1）的学习进度** ❌
2. 然后将**勾选的等级**标记为 100% 已完成 ✅
3. **未勾选的等级会变成 0%** ❌

### 不可逆操作
- 所有学习历史记录将被清空
- `last_seen_on`、`mastered_on`、`seen_count` 全部重置
- 无法恢复之前的进度

## 🎨 UI 改进

```

│ • 然后将勾选的等级标记为100%已完成   │
```

## 📊 对用户行为的影响

### 推送逻辑
"一键填满"会直接影响单词页和语法页的内容推送：

| 操作 | 单词页推送 | 语法页推送 |
|------|-----------|-----------|
| 勾选 N5、N4 | 只推送 N3、N2、N1 | 只推送 N3、N2、N1 |
| 勾选"全部单词" | 不推送任何单词 | 不影响语法 |
| 什么都不勾选 | 清空所有进度，从零开始 | 清空所有进度，从零开始 |

### 使用建议

✅ **正确用法**：
```
场景：你在其他平台已经学完了 N5 和 N4
操作：勾选"单词：N5、N4"和"语法：N5、N4"
结果：这两个等级标记为已完成，应用只推送 N3-N1 的内容
```

❌ **危险用法**：
```
场景：随便点击测试
操作：什么都不勾选就点"确认执行"
结果：所有学习进度被清空！
```
